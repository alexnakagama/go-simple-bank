# Go Simple Bank

A robust, modular, and production-ready RESTful API for simple banking operations, built with Go, PostgreSQL, and Gin. This project demonstrates best practices in Go architecture, configuration management, database migrations, and testing.

## Tech Stack

- **Language:** Go 1.25+
- **Web Framework:** [Gin](https://github.com/gin-gonic/gin)
- **Database:** PostgreSQL 15+
- **Database Driver:** [lib/pq](https://github.com/lib/pq)
- **Configuration:** [Viper](https://github.com/spf13/viper)
- **SQL Code Generation:** [sqlc](https://github.com/kyleconroy/sqlc)
- **Migrations:** [golang-migrate](https://github.com/golang-migrate/migrate)
- **Testing:** Go test, [Testify](https://github.com/stretchr/testify)
- **Containerization:** Docker, Docker Compose

---

## Architecture Overview

- **Layered Clean Architecture:**  
  - `api`: HTTP handlers, request validation, and routing.
  - `db/sqlc`: Data access layer, generated by sqlc from raw SQL queries.
  - `config`: Centralized configuration management.
  - `util`: Utility functions (e.g., random data generation).
- **Configuration via `.env`** using Viper.
- **Database migrations** and versioning.
- **Testable and modular**: All layers are independently testable.

## Getting Started

### Prerequisites

- Go 1.25+
- Docker & Docker Compose
- PostgreSQL client tools (`psql`, `createdb`, `dropdb`)

### Clone the repository

```sh
git clone https://github.com/alexnakagama/go-simple-bank.git
cd go-simple-bank
```

### Setup Environment

Copy `.env` or edit as needed:

```env
DB_PORT=5432
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=secret
DB_NAME=root
DB_SOURCE=postgresql://root:secret@localhost:5432/root?sslmode=disable
DB_DRIVER=postgres
SERVER_ADDRESS=0.0.0.0:8080
```

## Database Migrations

### Using Docker Compose

Start the database:

```sh
docker-compose up -d
```

### Run Migrations

```sh
make migrate
```

- Migration files are in `internal/db/migrations/`.
- You can create/drop the database with `make createdb` / `make dropdb`.

---

## SQL Code Generation

- SQL queries are in `internal/db/query/`.
- Run `sqlc generate` to regenerate Go code in `internal/db/sqlc/`.

## API Endpoints

### Accounts

- `POST /accounts` — Create a new account
- `GET /accounts/:id` — Get account by ID
- `GET /accounts` — List accounts (pagination: `page_id`, `page_size`)

#### Example: Create Account

```sh
curl -X POST http://localhost:8080/accounts \
  -H "Content-Type: application/json" \
  -d '{"owner": "alice", "currency": "USD"}'
```

#### Example: Get Account

```sh
curl http://localhost:8080/accounts/1
```

#### Example: List Accounts

```sh
curl "http://localhost:8080/accounts?page_id=1&page_size=5"
```

## Testing

Run all tests:

```sh

```

or

```sh
go test -v ./...
```

- Tests use the same `.env` configuration.
- Main test setup is in `internal/db/sqlc/main_test.go`.
